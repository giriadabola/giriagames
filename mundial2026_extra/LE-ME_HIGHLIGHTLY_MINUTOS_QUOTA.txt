Correção: Highlightly para minutos live + poupança de requests

O que foi feito:
1. A chave Highlightly foi configurada em script.js:
   const HIGHLIGHTLY_KEY = '5bc92d9d-077d-4e4c-be4a-403ae7f5792a';

2. A app tenta obter minutos diretos por esta ordem:
   - API-Football
   - Highlightly
   - football-data.org
   - AllSportsAPI
   - SofaScore proxy, se estiver configurado
   - ESPN proxy, se estiver configurado

3. Poupança de requests:
   Se uma API devolver minuto direto para um jogo live, a app NÃO chama as APIs seguintes de minutos nesse ciclo.

4. Dentro da própria Highlightly:
   A app tenta primeiro /matches/live.
   Se esse endpoint já trouxer minuto direto, não chama os endpoints seguintes da Highlightly nesse ciclo.

5. Regra visual mantida:
   Se o jogo estiver live mas nenhuma API devolver minuto direto, aparece só a bola vermelha a piscar, sem texto.

6. Regra Firebase mantida:
   Se o kickoff no Firebase ainda estiver no futuro, a app não grava 0-0 nem estado live/finished nesse jogo.

Nota:
A key ficou exposta no frontend. Para um projeto público, o ideal é usar proxy/backend ou gerar uma key nova depois dos testes.
