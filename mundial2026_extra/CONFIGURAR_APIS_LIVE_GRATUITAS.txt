CONFIGURAR APIS LIVE GRATUITAS / FREE TIER

A app já tenta usar várias fontes em cascata:

1. API-Football / API-Sports
   Campo no script.js:
   const API_FOOTBALL_KEY = 'COLOCA_AQUI_A_TUA_KEY';

2. football-data.org
   Campo no script.js:
   const FOOTBALL_DATA_KEY = 'COLOCA_AQUI_A_TUA_KEY';

3. Highlightly
   Campo no script.js:
   const HIGHLIGHTLY_KEY = 'COLOCA_AQUI_A_TUA_KEY';
   Se o fornecedor te der uma base URL diferente, muda também:
   const HIGHLIGHTLY_BASE = 'https://api.highlightly.net/football';

4. AllSportsAPI
   Campo no script.js:
   const ALLSPORTS_KEY = 'COLOCA_AQUI_A_TUA_KEY';

5. worldcup26.ir
   Não precisa de key.

6. TheSportsDB v1 free
   Não precisa de key adicional; usa a key pública 123.

ORDEM DE PRIORIDADE PARA DADOS LIVE
- API-Football
- football-data.org
- Highlightly
- AllSportsAPI
- worldcup26.ir
- TheSportsDB
- matches.json/Firebase como fallback visual

REGRA DE SEGURANÇA
A app só grava resultado final no Firebase se uma fonte real devolver estado final, como:
FINISHED, FT, Match Finished, AET, PEN, etc.

A app nunca grava resultado final apenas pelo horário local estimado.
Se o tempo for estimado, o cartão mostra só o ponto vermelho, sem minuto nem “Intervalo”.


11'S / LINEUPS
Ao clicar no card do jogo, a app tenta carregar os 11’s por esta ordem:
1. API-Football, se tiver fixture id e key configurada
2. Highlightly, se tiver match id e key configurada
3. AllSportsAPI, se tiver event id e key configurada
4. TheSportsDB, se tiver idEvent

Se nenhuma fonte tiver dados, aparece apenas “11’s não oficiais ainda indisponíveis”.
A interface não mostra nomes técnicos das fontes aos utilizadores.
