Correção aplicada: jogo futuro já não é gravado como 0-0 live.

Causa do erro:
O script misturava uma API que devolvia marcador 0-0 para jogos agendados com a janela de horário local `isMatchInLiveWindow()`.
Depois `syncFinishedApiResultsToFirebase()` gravava esse jogo em `worldcupextraMatches` como se estivesse live.

Regras novas:
1. Horário local NÃO cria live.
2. Jogo futuro/agendado NÃO grava 0-0.
3. Só grava live no Firebase quando uma API confiável disser que está live E trouxer minuto/estado direto.
4. Se já existir falso 0-0 live antigo, o script tenta limpar e voltar para scheduled com homeGoals/awayGoals null.
5. Sem Date.now() para inventar minuto.
